# Busyworld — Hand-off Guide

A complete orientation for anyone (human or a fresh AI session) picking up this
project. Read this first, then `docs/ARCHITECTURE.md`.

## 1. What this is
A living 2D pixel-art town (Godot 4.3) whose citizens are driven by local LLMs
(Ollama) via a Python "brain" server. Citizens perceive an embodied world, act,
talk, remember (SQLite, so they persist forever), have needs (hunger/coins), form
relationships, go indoors, and take on the founder's requests for a coin reward.
The long-term goal: grow to many towns/cities, give citizens real eyesight, and
let them run a real money-making venture from a building in town.

## 2. Current state (what's done)
- **Town**: generated by `tools/build_town.py` → bakes `godot/assets/ground/
  town_ground.png` + writes `godot/data/town_layout.json`. Plaza/fountain, civic
  row, shop row, corner workplaces, homes, tree-lined streets, day/night, parked
  side-view cars, sidewalk spurs to every door.
- **Citizens**: 12 in `godot/data/citizens.json` (shared by Godot + brain).
  Animated (idle/walk × 4 dirs) from LimeZu premade sheets, A* pathfinding,
  nameplates, speech bubbles, ground shadows, configurable count (`--agents N`).
- **Interiors**: click a building → enter a furnished pre-made room with the
  citizens inside; animated doors; "Leave" button. Citizens actually go indoors
  to work/rest.
- **Brain**: `brain/server.py` WebSocket middleware → per-citizen Ollama endpoints
  (`brain/agents.yaml`). Perception→prompt→action→memory→incentives in
  `brain/mind.py`. Heuristic fallback when models are down. Needs (hunger/food),
  relationships, nightly reflection, founder grants (food/coins).
- **Eyesight**: Godot renders a vision-citizen's POV (SubViewport) → base64 PNG →
  brain → vision model. Off by default (set `vision: true` per citizen).
- **Persistence**: `brain/memory.py` (SQLite at `brain/state/<world>.sqlite`).
- **Docs**: README, ARCHITECTURE, ROADMAP, WORLD_GUIDE, MODELS, this file.

## 4. How to run & verify (this is the important part)
The environment has **no GPU/display**, but Godot 4.3 renders fine under Xvfb with
software GL, and `tools/run_godot.sh` wraps this. **You can take real screenshots —
use them to verify every visual change.** The Godot binary lives at
`tools/bin/godot` (downloaded, git-ignored; re-download from the Godot 4.3 release
if missing).

```bash
# 1. regenerate the town after editing tools/build_town.py
python3 tools/build_town.py            # also writes tools/_montages/town_preview.png

# 2. import assets into Godot (needed after new/changed assets)
bash tools/run_godot.sh import

# 3. screenshot the running town (autopilot = no brain needed)
bash tools/run_godot.sh shot tools/screenshots/out.png 6           # -> out_00.png
bash tools/run_godot.sh shot tools/screenshots/out.png 6 --overview # whole map
#    debug flags (pass after the delay): --interior "Bakery", --povtest baker

# 4. full integration (brain + town)
python3 brain/server.py --world test --port 8765 &
bash tools/run_godot.sh series tools/screenshots/g.png 1 8   # connects to the brain
```

**Fast visual iteration trick:** `tools/build_town.py` also renders
`tools/_montages/town_preview.png`, a Python composite of the exact same layout
data Godot uses — iterate on the town there (seconds) before opening the engine.
Read any PNG with the Read tool to view it.

Python deps for the brain: `pip install -r brain/requirements.txt`
(websockets, aiohttp, pyyaml; plus pillow + numpy for the tools).

## 5. Key conventions / gotchas
- **Citizen sprite slicing** (`CharacterFrames.gd`): LimeZu premade sheets are 48px
  wide, character spans TWO grid rows; pair 1 = idle, pair 2 = walk; direction
  order in a row is Right, Up, Left, Down. Verified — don't "fix" it.
- **Town is data-driven**: never hand-place sprites in Godot. Edit `build_town.py`
  (the single source of truth) and regenerate.
- **Coordinates**: props/buildings are positioned by their baseline (bottom-centre)
  for Y-sorting. Interiors live in an off-map region the camera visits.
- **Authority split**: Godot owns space/time/positions; the brain owns
  mind/memory/economy. Perception is built by Godot; persistent stats (coins,
  hunger, goals, relationships) live in the brain's SQLite.
- **GL Compatibility** renderer is required for headless software rendering
  (set in `project.godot`). Don't switch to Forward+.
- **Godot CLI args** go after `--` (see `GameConfig._parse_cli`).
- `.godot/` cache and `brain/state/` are git-ignored; `.import` sidecars are
  committed (standard Godot workflow). First run re-imports.

## 6. The economy / money plan (per the founder)
Citizens need **food** and **virtual coins** to live; the founder supplies both
("give 30 food", "give 50 coins"). They earn coins by working, especially on the
founder's directives. The intended next step: a real **print-on-demand** venture
the citizens "work" on inside a building, earning real revenue that flows back as
treasury + incentives. Scaffold: `enterprises` table in `memory.py`, enterprise
detection in `server.py`. **No real financial integration is wired** — that needs
the founder's accounts/API keys.

## 7. Suggested next steps (priority order from the founder)
1. **Polish the town further** as needed (door alignment per building, more interior
   variety, a dedicated "ventures" workshop building).
2. **Deepen AI lives**: true two-party dialogues (the brain already gets who's in
   earshot), daily schedules, money pressure shaping behavior.
3. **Eyesight in practice**: test with a real vision model (`llama3.2-vision`,
   `llava`); tune POV size/zoom; let citizens act on what they see.
4. **Print-on-demand venture** (chosen): design a safe brain-side tool that lists/
   sells a product (e.g. Printful/Printify API), records real revenue → treasury,
   and pays incentives. Gate behind the founder's keys.
5. **Scale**: streaming map, a second town (`--world`), tiered minds.

See `docs/ROADMAP.md` for the fuller arc and `docs/MODELS.md` for model choices.
